### Amazon S3 Receipt Storage

### Objective
This page guides you on accessing the **Amazon S3** management interface in the AWS Console to verify private security settings, CORS rules, Lambda event notification triggers, and the actual upload folder structure for **FinVantage**.

### Overview
Amazon S3 provides highly durable and secure object storage. For FinVantage, S3 acts as the initial landing bucket for raw receipt image and PDF files uploaded from the Web frontend.

### Service Role in FinVantage
*   **Raw File Storage:** Stores raw uploaded receipt files securely. The bucket blocks all public access to safeguard user privacy.
*   **Presigned URL Mechanism:** Enables the React Frontend to upload files directly to S3 without routing payload data through Lambda backend memory and without opening public read/write permissions on the S3 bucket. Presigned URLs are generated by `finvantage-prod-importInvoice` with short expiration windows (e.g., several minutes).
*   **OCR Trigger Event:** When a new object lands in the `uploads/` prefix, S3 automatically fires an Event Notification triggering `finvantage-prod-ocrInvoice` Lambda function to extract text.

### Verification Steps on AWS Console

#### 1. Verify S3 Security Attributes

**Step 1:** Log in to AWS Console → Search `S3` → Select **S3**.

**Step 2:** Select the project bucket: `finvantage-invoices-hieu-2026-395840094907`.

**Step 3:** Open the **Permissions** tab:
*   Verify that **Block public access (bucket settings)** shows `Block *all* public access` set to **On**.

---

![Figure 5.4.1a. S3 Bucket Block all public access security settings.](../../../images/finvantage-s3-permissions.jpg)

---

**Step 4:** Switch to the **Properties** tab:
*   Scroll down to **Default encryption**. Confirm that status displays `Enabled` with Amazon S3 managed keys (SSE-S3) protecting data at rest.

#### 2. Verify CORS (Cross-Origin Resource Sharing) Configuration

**Step 1:** On the **Permissions** tab, scroll to the **Cross-origin resource sharing (CORS)** section.

**Step 2:** Verify the actual JSON configuration allowing Frontend integration:
```json
[
    {
        "AllowedHeaders": [
            "Content-Type",
            "x-amz-server-side-encryption"
        ],
        "AllowedMethods": [
            "PUT",
            "GET",
            "HEAD"
        ],
        "AllowedOrigins": [
            "https://main.dp5hgt6k889yu.amplifyapp.com"
        ],
        "ExposeHeaders": [
            "ETag"
        ]
    }
]
```
*Explanation:* This rule allows the Amplify Frontend domain to issue direct PUT (upload), GET, and HEAD HTTP requests to the S3 Bucket with secure headers.

---

![Figure 5.4.1b. CORS Configuration on S3 Bucket enabling direct Frontend integration.](../../../images/finvantage-s3-cors.jpg)

#### 3. Verify Event Notification Triggers

**Step 1:** Switch to the **Properties** tab, scroll down to **Event notifications**.

**Step 2:** Confirm event notification entry exists:
*   **Event types:** `s3:ObjectCreated:*` (Fires upon object creation).
*   **Prefix:** `uploads/` (Filters objects uploaded inside uploads folder).
*   **Destination:** Triggers Lambda function `finvantage-prod-ocrInvoice`.

---

![Figure 5.4.1c. Event Notification trigger configuration routing S3 uploads to OCR Lambda.](../../../images/finvantage-s3-trigger.jpg)

**Step 1:** Switch to the **Objects** tab.

**Step 2:** Click into the `uploads/` folder to inspect uploaded receipt image files from client sessions.
