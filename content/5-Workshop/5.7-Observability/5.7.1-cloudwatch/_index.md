### Amazon CloudWatch

### Objective
This page guides you on accessing **Amazon CloudWatch Console** to inspect detailed log entries in backend Lambda Log Groups, analyze runtime errors, and construct a centralized **CloudWatch Dashboard** for **FinVantage**.

### Overview
Amazon CloudWatch is AWS's centralized monitoring and management service. For Serverless applications like FinVantage, all runtime console outputs (`console.log()` or `console.error()`) generated by Node.js 24 Lambda functions are automatically collected and securely stored in CloudWatch Logs.

---

### Verification Steps on AWS Console

#### 1. Inspect Execution Logs (CloudWatch Logs)

**Step 1:** Log in to AWS Console ŌåÆ Search `CloudWatch` ŌåÆ Select **CloudWatch**.

**Step 2:** From the left menu, select **Logs** ŌåÆ Select **Log groups**.

**Step 3:** Enter the backend Lambda Log Group name (e.g., `/aws/lambda/finvantage-prod-analyzeInvoice`). Click into the Log Group.

**Step 4:** Inspect the **Log streams** list, confirming streams corresponding to Lambda execution sessions.

#### 2. Construct and Inspect CloudWatch Dashboard

Build a centralized Dashboard to track infrastructure health and API invocation metrics on a single screen.

**Step 1:** From the left menu of CloudWatch Console, select **Dashboards** ŌåÆ Click **Create dashboard**.

**Step 2:** Name the Dashboard `finvantage-prod-dashboard` ŌåÆ Click **Create**.

**Step 3:** Add key monitoring widgets to the Dashboard:
*   **Widget 1: Lambda Invocations**
    *   *Type:* Stacked area or Number widget.
    *   *Metric:* `AWS/Lambda` > `Invocations` (Select core functions: `importInvoice`, `ocrInvoice`, `analyzeInvoice`).
*   **Widget 2: Lambda Execution Duration**
    *   *Type:* Line chart widget.
    *   *Metric:* `AWS/Lambda` > `Duration` (Tracks AI invocation latency).
*   **Widget 3: API Gateway Requests & Errors**
    *   *Type:* Line chart widget.
    *   *Metric:* `AWS/ApiGateway` > `Count` and `5XXError`.
*   **Widget 4: RDS & Cache CPU Utilization**
    *   *Type:* Line chart widget.
    *   *Metric:* `AWS/RDS` > `CPUUtilization` and `AWS/ElastiCache` > `CPUUtilization`.

**Step 4:** Click **Save dashboard**.
