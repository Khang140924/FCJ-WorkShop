### Week 3 Objectives:

* Research the theoretical overview of the Amazon Elastic Compute Cloud (Amazon EC2) cloud computing service.
* Learn about Instance Types, server pricing models, and network security mechanisms via Security Groups.
* Learn about automated scaling and load balancing solutions: Auto Scaling Group and Elastic Load Balancing (ELB).
* Practice initializing, configuring, and managing the lifecycle of EC2 virtual servers on the console interface.

### Week 3 Achievements:

* Mastered the concept of Amazon EC2, clearly understanding how to select the optimal and most suitable Instance Type (in terms of CPU, RAM, Network) for various software application architectures.

* Clearly distinguished the operational mechanism of Security Groups (Instance-level virtual firewall - Stateful) to securely control inbound/outbound network traffic to the server.

* Understood the operational principles of Elastic Load Balancing (traffic distribution) and Auto Scaling Group (automatically scaling server capacity based on actual performance) to ensure High Availability for the system.

* Successfully configured and launched an Amazon EC2 Instance running the Amazon Linux 2 operating system.

* Accurately configured Key Pairs and used remote connection tools (such as SSH Terminal/PuTTY) to directly access and manage the virtual server from a personal computer.

* Successfully set up Security Group Rules, opening basic network ports (Port 22 for SSH, Port 80/443 for HTTP/HTTPS) to allow the server to communicate with the Internet environment.

* Practiced installing a basic test Web Server (Apache/Nginx) on EC2 using the User Data feature (running automated scripts upon server initialization).

* Clearly understood how a Backend application (Java, PHP, Go) is actually deployed and operated on a cloud server environment instead of a standard Localhost environment.

* Improved Linux operating system administration skills via the Command Line Interface (CLI) – a mandatory and extremely important skill for a professional Backend / Cloud Engineer.
